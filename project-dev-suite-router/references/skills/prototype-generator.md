# 原型生成 Skill

根据需求文档生成纯 HTML 静态原型页面。

## 触发场景

- 用户要求生成原型页面
- 用户说"帮我画个原型"、"做个页面 demo"
- 用户提供了需求文档并要求可视化

## 前置条件

- 需要有明确的功能需求或菜单结构
- 如果项目中已有需求文档，先读取需求文档

## 执行流程

### Phase 1: 确认原型范围

1. 确认要生成哪些页面（主架构页 + 子页面）
2. 确认终端类型：PC端 / H5移动端 / 两者都要
3. 确认交互深度：仅布局展示 / 包含基础交互（Tab切换、弹窗等）

### Phase 2: 参考检索（关键步骤）

在生成原型前，搜索参考资料以提升质量：

**搜索策略（按优先级）：**
1. **项目内已有原型**：扫描 `docs/prototypes/` 目录，复用已有布局模式
2. **联网搜索类似产品 UI**：搜索"{业务类型} UI design"、"{业务类型} dashboard template"
3. **GitHub 搜索开源模板**：搜索 "vue admin template"、"mobile ui template html" 等
4. **skills.sh 搜索**：运行 `npx skills find {相关关键词}` 看是否有现成 UI skill

**从参考中提取的模式：**
- 菜单组织方式（侧边栏/顶部Tab/底部导航）
- 列表交互模式（表格/卡片/时间线）
- 表单布局模式（弹窗/抽屉/独立页）
- 状态展示方式（标签/进度条/看板）
- 配色和视觉风格

### Phase 3: 生成主架构页

**PC端模板特征：**
- 左侧菜单栏（支持一级/二级菜单，点击切换 iframe 内容）
- 顶部导航栏（Logo + 用户信息）
- 右侧内容区（iframe 加载子页面）
- 最小宽度 1200px
- 菜单高亮当前选中项

**H5端模板特征：**
- 底部 Tab 导航（一级菜单，最多5个）
- 顶部标题栏 + 返回按钮
- 顶部二级 Tab（当前一级菜单下的子菜单）
- 内容区 iframe 加载子页面
- 视口宽度 375px 设计
- 手机框外壳包裹

### Phase 4: 生成子页面

每个功能模块生成独立 HTML 文件，必须包含：
- 真实业务字段（从需求文档提取）
- 模拟数据（中文，3-5条）
- 状态流转交互
- 筛选/搜索功能的 UI 展示

**通用组件库（内联CSS）：**
- 搜索栏 + 筛选条件
- 数据表格（带分页）
- 表单（输入框、下拉、日期选择、文件上传）
- 卡片列表（移动端优先）
- 弹窗/抽屉
- 状态标签（待处理/进行中/已完成/超时）
- 时间线、统计卡片、空状态提示

### Phase 5: 主架构与子页面联动

确保主架构页的菜单点击能正确加载对应子页面（相对路径 iframe）。

### Phase 6: 输出文件

```
docs/prototypes/
├── 售前原型-{项目名}-PC[主架构].html
├── 售前原型-{项目名}-H5[主架构].html（如需要）
└── pages/
    ├── 售前原型-{功能名1}.html
    └── 售前原型-{功能名2}.html
```

## 技术规范

### 设计规范
- 主色调：#1890ff | 成功：#52c41a | 警告：#faad14 | 错误：#ff4d4f
- 背景色：#f5f7fa | 文字：#333/#666/#999
- 圆角：4px/8px/12px | 阴影：0 2px 8px rgba(0,0,0,0.1)
- 字体：-apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif

### 交互规范
- 按钮 hover 变色、表格行 hover 高亮
- 弹窗有遮罩层 + 关闭按钮
- Tab 切换有选中态
- 状态标签用不同颜色区分

## 输出规则

- 每个 HTML 文件完全独立，不依赖外部 CDN
- 使用模拟数据填充（中文，贴近真实业务）
- 主架构页通过 iframe 加载子页面
- 文件编码 UTF-8


## V1.1 设计契约与状态覆盖

生成原型前，先读取 `docs/PRD.md`，并为有 UI 的项目创建 `docs/DESIGN.md`。DESIGN 至少包含：

- 视觉方向和参考来源。
- 色彩、排版、布局密度、圆角、阴影和图标规则。
- 导航、表单、卡片、列表、弹窗等组件的复用规则。
- 默认、悬停、选中、禁用、加载、空、错误和无权限状态。
- 明确需要避免的视觉或交互模式。

不要默认套用固定后台模板或固定配色。原型必须验证主流程和异常路径；若原型增加了 PRD 范围外的功能，先回到 PRD 确认。
