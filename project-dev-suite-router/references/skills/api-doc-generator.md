# API 接口文档生成 Skill

根据代码或需求自动生成标准化的 API 接口文档。

## 触发场景

- 用户要求生成接口文档
- 用户说"帮我写 API 文档"
- 项目代码已有 Controller/Router，需要补充文档

## 执行流程

### Phase 1: 识别接口来源

**方式A - 从代码提取**：
1. 扫描项目中的 Controller（Java）或 Router（Python/FastAPI）
2. 提取路径、方法、参数、返回值信息
3. 读取注释/docstring 作为描述

**方式B - 从需求文档生成**：
1. 读取需求文档中的接口清单
2. 根据数据模型推导请求/响应结构
3. 补充通用字段（分页、排序等）

### Phase 2: 生成接口文档

```markdown
# {模块名} 接口文档

## 基础信息
- 基础路径: `/api/v1/{module}`
- 认证方式: Bearer Token
- 内容类型: application/json

---

## 1. {接口名称}

**接口说明**: {一句话描述}

**请求信息**:
- 方法: `POST`
- 路径: `/api/v1/xxx`
- 权限: 需要登录

**请求头**:
| 参数 | 必填 | 说明 |
|---|---|---|
| Authorization | 是 | Bearer {token} |

**请求参数**:
| 参数 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| name | String | 是 | 名称，最长50字符 | "会议A" |
| status | Integer | 否 | 状态 1-启用 0-禁用 | 1 |

**请求示例**:
```json
{
  "name": "会议A",
  "status": 1
}
```

**响应参数**:
| 参数 | 类型 | 说明 |
|---|---|---|
| code | Integer | 状态码，200=成功 |
| message | String | 提示信息 |
| data | Object | 业务数据 |

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1
  }
}
```

**错误码**:
| 错误码 | 说明 |
|---|---|
| 400 | 参数校验失败 |
| 401 | 未登录 |
| 403 | 无权限 |
```

### Phase 3: 输出

- 保存到 `docs/api/` 目录
- 文件命名：`{模块名}-api.md`
- 如果模块较多，生成一个 `docs/api/README.md` 作为索引

## 通用规范

### 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null
}
```

### 分页响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 通用错误码
| 错误码 | 说明 |
|---|---|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 输出规则

- 所有输出使用中文
- 每个接口必须有请求示例和响应示例
- 参数说明必须包含类型、是否必填、约束条件
- 枚举值必须列出所有可选项
